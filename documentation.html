1. System Architecture

TinyCAD is a browser-based simple application. It leverages the HTML5 Canvas API for direct-to-buffer rendering, avoiding the overhead of DOM-based geometry.

The Object Model

Every entity in the workspace is stored in the lines[] array. A standard entity schema includes:

{
    x1, y1, x2, y2: number, // World coordinates
    type: "line" | "window" | "door",
    locked: boolean,        // Length constraint
    mirrored: boolean       // Door swing orientation
}

2. Coordinate Systems

The application maintains a separation between Screen Space (the pixels on your monitor) and World Space (the actual CAD plane).

$World_{pos} = \frac{Screen_{pos}}{Zoom} + Camera_{offset}$

This allow for infinite panning and zooming without losing coordinate precision. The Axis Indicator in the bottom right remains locked to screen space while accurately representing world-space orientations.

3. Drawing & Snap Logic

Vertex Snapping

To ensure structural integrity, the engine performs a proximity check on every mousedown event. It calculates the Euclidean distance between the cursor and all existing vertices:

$d = \sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}$

If $d < 15px$, the cursor "snaps" to the vertex, ensuring walls and doors connect perfectly.

4. Architectural Features

Window Rendering

Windows are generated using a perpendicular offset. The engine calculates the angle of the base line and offsets two parallel lines to create the frame symbol.

Door Flip/Mirror New

The Mirror feature uses a mathematical sign-multiplier. When mirrored is toggled, the arc sweep and leaf rotation angles are inverted:

  • Standard: Arc sweep calculated at $\theta - 90^\circ$
  • Mirrored: Arc sweep calculated at $\theta + 90^\circ$

5. Parametric Drafting

The Parametric Panel (found in the Inspector) allows for rapid input via compass directions.

CommandEffectAxis
NSubtract from YNorth
SAdd to YSouth
EAdd to XEast
WSubtract from XWest

Example: N200 draws a line 200 units Up from the last point.

6. Export Engine

TinyCAD uses a Boundary-Aware Export logic. Instead of capturing the screen, it calculates the Axis-Aligned Bounding Box (AABB) of your drawing and crops the output accordingly.

DXF Inversion: Note that DXF exports automatically invert the Y-axis. This is to ensure compatibility with AutoCAD's Cartesian system where "Up" is positive Y.
